# EventSpace 页面线框与信息层级 v1

> 文字线框用于确认页面骨架和内容顺序。它不表示最终像素视觉稿，也不包含功能代码。

## 1. 路由总览

| 区域 | 主要页面 | 目的 |
| --- | --- | --- |
| 公开 | 首页、邀请进入、认证、法律页 | 解释产品、创建/加入、登录。 |
| 用户首页 | 进行中、已归档、搜索/筛选、账户菜单 | 回到自己的活动空间。 |
| 创建 | 分步创建向导、成功/分享 | 快速建立私密临时房间。 |
| 房间 | Chat、Board、Itinerary、成员、分享、管理面板 | 活动进行中的核心协作。 |
| 归档 | 回忆册入口、只读 Chat/Board/Itinerary、个人移除确认 | 浏览自己的只读活动记忆。 |
| 状态 | 审核等待、冻结、处理归档、访问终止、断线 | 清晰说明当前可否操作。 |

## 2. 公开首页

### 移动端

```text
┌──────────────────────────────────┐
│ EventSpace                  Log in│
│                                  │
│   [抽象图钉照片 / UI 卡片背景]     │
│                                  │
│   Make the moment                │
│   live longer.                   │
│                                  │
│   [ Create a room              ] │
│   [ Join with a link           ] │
│                                  │
│   A private space for the moment.│
└──────────────────────────────────┘
│ Chat      Board       Itinerary   │
│ 三张简短功能卡，随后是法律链接        │
└──────────────────────────────────┘
```

- 首屏占满首个视口；背景照片/模拟 UI 是抽象层，不承载任何可读信息。
- `Create a room` 是唯一强主按钮。`Join with a link` 为次按钮。
- 不展示公开房间、活动发现流、价格比较或冗长营销章节。

### 桌面端

- 保留同一阅读顺序，增加左右留白与背景卡片的空间层次。
- 文字列限制在舒适阅读宽度，避免标题横跨整屏。
- 功能卡在首屏之后以三列呈现。

## 3. 用户首页

### 默认：单张横滑房间流（移动端）

```text
┌──────────────────────────────────┐
│ EventSpace                    ◉   │
│ [ Search rooms...              ]  │
│                                  │
│ Active              Archived      │
│                                  │
│     ┌────────────────────────┐   │
│     │                        │   │
│     │  75% 画板缩略图          │   │
│     │  （完整画板默认封面）     │   │
│     │                        │   │
│     ├────────────────────────┤   │
│     │ Summer at the lake  ▯  │   │
│     │ • 3h 12m left · 6 people│  │
│     └────────────────────────┘   │
│              ●  ○  ○              │
│                                  │
│                         [ + Create]│
└──────────────────────────────────┘
```

- 横滑一次突出一张大卡；收藏书签浮于卡片右上角。
- 进行中状态及倒计时仅位于信息区，绝不压在画板缩略图上。
- 搜索与筛选展开后只改变结果集；仍以单张横滑为默认浏览方式。
- 用户可切换到双列纵向浏览，适合快速查找；该模式不取代默认阅读体验。
- 空状态只保留一句情绪化短句和创建按钮。

### 桌面端

- 活动卡按可用空间扩展为 2–3 列，但卡片仍保持“封面 + 信息区”的杂志比例。
- 搜索、筛选与视图切换占据标题行右侧，不形成数据后台式侧栏。

## 4. 创建房间向导

```text
┌──────────────────────────────────┐
│ ←  Create a room           2 / 5 │
│ ────────────────                   │
│                                  │
│ How should this room be led?     │
│                                  │
│ ┌──────────────────────────────┐ │
│ │ Host-led                      │ │
│ │ A host and admins guide it.   │ │
│ └──────────────────────────────┘ │
│ ┌──────────────────────────────┐ │
│ │ Community-led                 │ │
│ │ Members decide by vote.       │ │
│ └──────────────────────────────┘ │
│                                  │
│ [ Continue                       ]│
└──────────────────────────────────┘
```

- 一步只完成一个决策；顶部提供返回、标题、低干扰进度 `n / 5` 与细线。
- 每项采用大号说明卡，不使用下拉框：名称/说明、模式、时长/人数、进入方式、可选高级设置。
- `More settings` 收纳审核和不常用配置。
- 付费只在创建时可选，或配额超出时再出现；不在日常导航中常驻。

## 5. 邀请、身份与审核等待

### 邀请页

```text
┌──────────────────────────────────┐
│ EventSpace                        │
│                                  │
│          Summer at the lake       │
│          Ends in 3h 12m           │
│          Up to 10 people          │
│          One line of description  │
│                                  │
│ What's your name?                 │
│ [                ]                │
│ Choose an avatar                  │
│ [ Join this room                  ]│
└──────────────────────────────────┘
```

- 只展示名称、说明、结束时间和人数上限。
- 不显示聊天、画板、完整成员名单或其他私密内容。
- 补全身份/登录使用全屏认证页，保留房间名称作上下文。

### 审核等待

- 保持邀请卡的房间信息；显示安静的等待反馈和可退出入口。
- 不提前加载房间内容。

## 6. 房间共同外壳

```text
┌──────────────────────────────────┐
│ [←]            Summer at the lake │
│        • 3h 12m       Share   More │
│ ──────────────────────────────── │
│      Chat      Board    Itinerary │
│ ──────────────────────────────── │
│                                  │
│             当前页面内容           │
└──────────────────────────────────┘
```

- 顶栏左侧显示返回 Rooms 的图标，不显示当前页面指示图标；房间名、状态和倒计时按整个屏幕严格居中。
- 房间名栏下方常驻 Chat / Board / Itinerary 导航。仅通过按钮切换，不使用左右滑动手势；三个页面保持挂载，以保留草稿、局部选择与滚动位置。
- 最后 15 分钟提升细提示的视觉权重；最后一分钟轻微提示，绝不使用遮挡弹窗。
- 分享、成员与更多操作进入抽屉/底部面板，不把管理工具常驻在正文。

## 7. Chat

```text
┌──────────────────────────────────┐
│ [Pinned: Bring a jacket.      × ] │
│                                  │
│      Maya joined the room         │
│                                  │
│ ○ Maya                            │
│   Meet by the entrance at 7?      │
│   6:42 PM                         │
│                                  │
│                  Yes, see you.    │
│                         6:43 PM   │
│                                  │
│ [ Replying to Maya             ]  │
│ [ Message...       ] [voice] [↑]  │
└──────────────────────────────────┘
```

- 连续消息只在首条展示头像与昵称；系统消息使用居中小字与极弱分隔。
- 输入栏固定底部；文字发送与按住说话是独立操作。
- 置顶消息为可收起的顶部窄卡。
- 投票、行程变更等以嵌入卡片出现；结束投票只保留问题、票数、结果、结束时间。
- 禁言状态保留输入区域但以说明文案替代可编辑控件。

## 8. Board

```text
┌──────────────────────────────────┐
│         [ Board ]  [ Sequence ]  │
│                                  │
│       ┌──────────────┐           │
│       │  photo + pin  │           │
│       └──────────────┘           │
│  ┌─────────────┐                  │
│  │ photo + pin │    note          │
│  └─────────────┘                  │
│                         [ + ]     │
│                    [ ⌂ overview ]│
└──────────────────────────────────┘
```

- 初始是中心空白画布，内容自然散布；可用空间有限，越界自动吸附至合法位置。
- 照片固定微倾斜，顶部白条与中央图钉；不使用纹理、渐变或拟物背景。
- 选中本人内容才出现浮动工具条；右下角保留上传入口与“回到全览”。
- 单击照片进入详情层：移动端评论从底部上推，桌面端评论固定在大图右侧。

### Sequence

- 两列纵向照片流，按上传时间排序；顶部固定 `Back to board`。
- 点击照片仍进入同一详情层。

```text
│ Today                             │
│ ┌──────────────────────────────┐ │
│ │ 7:00 PM · Meet at Riverside  │ │
│ │ In progress                   │ │
│ │ Maya · 4 attending      ›     │ │
│ └──────────────────────────────┘ │
│                                  │
│ Tomorrow                          │
│ ┌──────────────────────────────┐ │
│ │ 10:00 AM · Brunch             │ │
│ │ Not started              ›     │ │
│ └──────────────────────────────┘ │
└──────────────────────────────────┘
```

- 按时间纵向排序，跨日用简洁日期文字分隔。
- 时间卡优先显示时间、标题、地点、负责人、状态和参与人数。
- 地点仅用文字和 Google Maps / Apple Maps 外链图标；第一版不嵌入真实地图。
- 状态用文字与极细状态点，不使用大面积色块。

## 10. 归档处理、归档详情与访问终止

### 归档处理

```text
┌──────────────────────────────────┐
│                                  │
│         The moment is being       │
│              preserved.           │
│                                  │
│         Saving your room…         │
│                                  │
│      You will be notified here.   │
└──────────────────────────────────┘
```

- 完全只读；状态清晰但安静，不让用户误认为资料遗失。

### 归档详情

- 首屏为大幅完整画板预览、名称、日期和简要活动信息；随后进入只读 Chat、Board、Itinerary。
- 用户可单独选择自己列表中的卡片封面；删除只影响自己的列表。

### 被踢出/拉黑

- 仅显示无内容的访问终止页与必要说明；不泄露任何房间信息。

## 11. 抽屉、底部面板与提示

- 分享、成员、审批、评论、投票发起、不可逆操作在移动端使用底部面板，桌面端可转为右侧抽屉。
- 高风险操作必须单独二次确认，并用明白语言说明后果。
- 普通错误、断线和重试只用短底部提示条与局部按钮，不以中心弹窗打断聊天。
- PWA 在首访以轻量提示出现；不干扰核心 CTA 与邀请进入流程。
## 2026-07-18 当前同步：页面结构变更

本文件的 v1 线框仍作为早期页面层级参考；当前正式 mock 已有以下结构变化：

- `/rooms` 顶部筛选改为当前筛选项按钮，支持 All / Active / Achieved / Favorite，下拉展示其他筛选；旁边有 Edit 文本按钮。
- `/rooms` 卡片默认不显示收藏，进入 Edit 后左上角显示 Favorite，右上角显示红色删除按钮。
- `/rooms` 卡片主视觉必须展示真实 Board snapshot，而不是房间名或固定 mock 画板。
- `/rooms/new` Step 3 改为小时/分钟两个纵向滚轮；完成页保留邀请卡片、Save card、Open this room。
- `/rooms/[roomId]` 顶部主 tab 保持 Chat / Board / Itinerary；Board tab 可展开 Board / Sequence 子选项，画板内部不再放独立切换条。
- Chat 输入栏固定底部：左侧加号工具托盘，输入框内部右侧语音按钮，右侧发送按钮。
- Chat Poll 创建器为底部覆盖层中的横滑卡片；Votes 历史为全屏暗色覆盖层。
- Board 右下角工具为相机、相册、文本、画线、画布背景等独立图标；画布背景选择在工具上方弹出。
- Board 文本和涂鸦编辑不再使用传统表单页，而是移动端优先的覆盖层/全屏画板。
- Board 点击空白区域结束当前编辑状态；删除按钮位于画板右上角。
