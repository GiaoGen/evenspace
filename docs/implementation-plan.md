# EventSpace 实施任务顺序

> 本文是执行顺序与验收门槛，不替代产品规格。未通过每阶段的验收，不进入后续依赖阶段。

## Phase 0 — 视觉与交互定稿

1. 确定公开首页、创建房间、房间内三页、归档页的视觉方向与信息层级。
2. 定义明亮/深色设计 token：颜色、字体、栅格、圆角、阴影、间距、焦点状态、动效时长与缓动。
3. 明确移动端与桌面端的关键断点、顶部导航、画板手势和加载/空状态。
4. 输出可审阅的页面线框与高保真视觉规则。

验收：产品负责人确认布局、视觉语言和关键交互；不创建功能页面。

## Phase 1 — 工程基础与质量门槛

1. 建立 App Router 路由组、组件目录、共享类型、环境变量模板与配置校验。
2. 加入 Tailwind、组件基础、动效库、Supabase SSR 客户端和测试工具。
3. 设置 ESLint、格式化、类型检查、单元/端到端测试骨架及 CI。
4. 定义性能、错误处理、日志脱敏和 Chromium/Safari 浏览器测试基线；无障碍设计留待核心体验稳定后单独评估。

验收：空项目可通过 lint、类型检查、构建与基础测试；不含真实凭据。

## Phase 2 — 身份、成员资格与数据库安全

1. 创建 Supabase schema、迁移、枚举、审计事件和 RLS 策略。
2. 接入匿名访客、邮箱 magic link/OTP、Google 登录与访客身份认领。
3. 实现房间成员、角色、昵称唯一性、审核进入、链接/邀请码轮换和设备级拉黑。
4. 以测试验证每个表的未授权读取/写入均被拒绝。

验收：访客和登录用户只能访问授权房间；服务角色密钥没有出现在客户端；RLS 测试通过。

## Phase 3 — 房间生命周期与基础界面

1. 实现创建、进入、离开、Host 转让、管理员、禁言、踢出和拉黑。
2. 实现服务器时间驱动的到期校验、手动结束、归档状态机与到期通知。
3. 建立首页的进行中/归档卡片、房间顶部信息栏和顶部三项导航。
4. 建立加载、断线、空状态、权限拒绝和归档中的体验。

验收：到期后所有写操作从服务端被拒绝；成员权限变化即时生效；手机与桌面可用。

## Phase 4 — 聊天、行程与投票

1. 实现文字、短语音、回复、@、撤回、置顶、搜索、系统消息及频率限制。
2. 实现行程、负责人、参与状态、人数上限、地点卡片、时区和提醒。
3. 实现 Host/无 Host 投票，含固定分母、公开/匿名、截止与自动结算。
4. 建立私有 Realtime channel，并对断线重连和事件幂等性做测试。

验收：数十名模拟成员可实时同步；投票与行程权限遵循产品规则；无未授权广播。

## Phase 5 — 画板与媒体

1. 实现私有媒体上传、浏览器压缩、EXIF 清除、服务端文件验证、缩略图和配额。
2. 实现自由画板、图钉照片视觉、固定微倾斜、边界/遮挡检测、自动吸附和顺序显示。
3. 实现内容所有权、评论/表情、两分钟撤回、图片保存及私有签名 URL。
4. 优化移动触控、桌面拖动、120Hz 设备动效与大画板渲染。

验收：媒体不可公开猜测访问；发布者外无人可编辑；画板在目标浏览器保持顺滑。

## Phase 6 — 归档、付费与外部服务

1. 完成只读个人归档、免费 30 天删除、备份清理、永久归档和容量扩展。
2. 接入 Stripe 一次性付款与签名 webhook；所有权益以服务端记录为准。
3. 接入 Google Places、Google/Apple Maps 外跳、API 限额与费用监控。
4. 配置 Resend SMTP、浏览器推送、支持联系入口和交易/归档通知。

验收：付款重放、伪造 webhook、过期归档与未授权下载均被拦截；外部 API 具有预算上限。

## Phase 7 — PWA、合规与发布

1. 实现 PWA manifest、安装体验、仅应用壳缓存与草稿恢复。
2. 完成隐私政策、条款、社区规则、Cookie 说明、年龄确认和数据主体请求流程。
3. 在 Chrome、Edge、Safari（macOS/iOS/iPadOS）进行响应式、无障碍、暗色模式、低网速和长时活动测试。
4. 进行安全审查、依赖扫描、备份恢复演练、支付与邮件生产环境检查。

验收：发布检查清单全部通过；法律文本经审阅；没有关键安全或数据删除缺陷。
