# EventSpace 设计系统规范 v1（待视觉稿复核）

> 范围：页面视觉、排版、组件形状与性能边界。它不定义业务逻辑；在高保真稿复核前，不进入功能代码实现。

## 1. 设计原则

1. **活动感而非 SaaS 感**：每个房间是短暂活动的容器，首页和归档应像回忆册，而不是数据后台。
2. **高级但不厚重**：暖白、墨黑、低饱和灰阶、强留白、杂志式标题；不引入品牌彩色系统。
3. **信息永远先于装饰**：卡片、阴影和动态用于建立层级与反馈，不能遮挡内容或拖慢操作。
4. **移动优先、桌面扩展**：移动端保持单一阅读重心；桌面端只增加空间与辅助栏，不复制复杂后台布局。
5. **性能是视觉质量的一部分**：禁止渐变、毛玻璃/高斯模糊、纹理、大面积滤镜和会持续触发重排的动效。

## 2. 字体与加载预算

| 角色 | 方案 | 使用范围 |
| --- | --- | --- |
| 界面无衬线 | Geist Variable（Latin 子集） | 正文、表单、聊天、时间、按钮、信息密集界面。 |
| 展示衬线 | Bodoni Moda Variable（Latin 子集；仅必要字重） | 公开首页主标题、归档回忆册标题、少量大标题与 EventSpace 字标。 |
| 回退 | `ui-sans-serif, system-ui, sans-serif` / `Georgia, serif` | 字体未到达时立即显示，保证首屏不等待。 |

- 两套字体均使用 Next.js 字体优化并从本站静态资源提供；浏览器不在运行时请求 Google 等第三方字体域名。
- 展示衬线不应用于聊天、表单、行程、投票和成员列表。
- 发布前检查字体传输体积；展示字体仅保留 Latin 子集与设计实际使用的字重。若达不到首屏性能预算，先缩减展示字重，再考虑更换字体，不牺牲聊天和画板性能。

### 深色模式

| Token | 建议值 | 用途 |
| --- | --- | --- |
| `canvas` | `#12110F` | 页面背景。 |
| `surface` | `#1B1916` | 主卡片、面板、输入区。 |
| `surface-muted` | `#292621` | 次级卡片、选中后的弱背景。 |
| `ink` | `#F5F1EA` | 主文字、反转主按钮。 |
| `ink-muted` | `#B9B2A9` | 次要文字与图标。 |

## 4. 尺度、形状与阴影

| 项目 | 规范 |
| --- | --- |
| 间距基线 | 4px；常用间距为 8、12、16、24、32、48、64px。 |
| 大卡片圆角 | 24px。 |
| 次级卡片/底部面板 | 20px。 |
| 输入、标签、小工具 | 14–16px。 |
| 主按钮 | 18px；移动端最小高度 48px。 |
| 内容最大宽度 | 常规页面 1240px；阅读/认证页 560px；房间聊天主列 760px。 |
| 卡片边框 | 1px `line`，仅在阴影不足以分层时使用。 |
| 阴影 | 静态、低透明度、多层柔影；悬浮或按下只调整强度和 1–2px 位移，不动画模糊半径。 |

## 5. 排版层级

| 层级 | 字体 | 建议尺寸与行高 | 场景 |
| --- | --- | --- | --- |
| Display | Bodoni Moda | 移动 48/0.98，桌面 80/0.94 | 首页情绪化标题。 |
| H1 | Bodoni Moda | 36/1.05 | 归档回忆册、少量页面标题。 |
| H2 | Geist 600 | 24/1.2 | 区块标题、房间标题。 |
| Body | Geist 400 | 16/1.55 | 正文、消息。 |
| Small | Geist 500 | 13/1.35 | 时间、状态、说明。 |
| Label | Geist 600 | 12/1.2，适度字距 | 导航与小标签。 |

- `EventSpace` 字标使用 Bodoni Moda 的较重字重与紧凑字距，第一版不设计独立 Logo 图形。

## 6. 动效 token 与性能边界

| 类型 | 时长 | 曲线 | 用途 |
| --- | --- | --- | --- |
| 点击/按压 | 120–160ms | 轻快 ease-out | 按钮、图标、卡片按下。 |
| 面板/抽屉 | 220–280ms | 轻弹性 cubic-bezier | 底部分享卡、评论区、更多菜单。 |
| 页面/标签切换 | 180–220ms | 克制 ease-out | Chat、Board、Itinerary。 |
| 首屏入场 | 360–520ms | 分时缓出 | 抽象照片与 UI 卡片轻微上移后定格。 |
| 主题切换 | 200ms | linear/ease | 仅颜色，不让内容重入场。 |

- 只动画 `transform` 与 `opacity`；滚动、拖动、缩放中的元素不执行阴影、滤镜或布局动画。
- 120Hz 设备以接近 120fps 为目标；普通设备优先稳定输入、滚动和拖动。
- 画板使用 DOM/CSS transform 与合成层策略，不使用重量级 Canvas 或 WebGL 渲染器。

## 7. 页面骨架

### 公开首页

全视口首屏：抽象图钉照片与 UI 卡片作背景，前景为情绪化标题、`Create a room` 主按钮、`Join with a link` 次按钮。右上角仅保留登录和低调菜单。首屏下方以三张精致卡片介绍 Chat、Board、Itinerary。

### 用户首页

公开 Landing 只属于未登录入口，不进入登录后的导航层级。登录后的 Rooms 首屏直接从“进行中 / 已归档”切换与房间卡片开始，不显示营销式欢迎标题。默认进行中；移动端以单张大卡片横滑展示，画板缩略图约占 75%，信息置于底部。搜索、收藏和筛选不改变横滑阅读结构；另提供双列纵向浏览。

### 房间内

房间使用两层实体顶部结构：第一层左侧为返回 Rooms 的圆形图标，中间为严格居中的房间名与状态/倒计时，右侧为分享、成员和更多菜单；不显示当前页面指示图标。第二层常驻 Chat / Board / Itinerary 三项按钮导航，使用细底线表示当前页。页面只通过按钮切换，不绑定左右滑动手势；切换使用短促透明度过渡，并保持三个页面挂载以保存局部状态。聊天采用弱气泡分组与固定输入栏；画板内容从中心自然散开，选中后出现浮动工具条；行程采用按日期分隔的纵向时间卡。

### 归档

以“活动回忆册”作为入口：大画板预览和活动信息在前，再进入 Chat、Board、Itinerary。用户可为自己归档列表挑选照片封面，但默认封面是完整画板。

## 8. 已确认的非视觉边界

- 当前 Phase 0 不将无障碍设计设为页面视觉约束；此项在核心主体验稳定后再独立评估。
- 真实地图缩略图不纳入第一版，行程只提供地点文字和 Google Maps / Apple Maps 外链。
- 付费入口不常驻；只在创建时选择或超配额时呈现。
- PWA 首访显示一次轻量安装提示；浏览器是否给出原生安装请求取决于平台。
## 2026-07-18 当前同步：设计系统落地补充

当前实现继续保持低饱和、克制、移动端优先的方向，但以下组件规则已经在正式 mock 中形成，应作为后续实现参考：

- 字体：生产构建使用 `public/fonts` + `@font-face`，保留 Geist 正文字体和 Bodoni Moda 衬线标题，不再依赖 `next/font` 远程下载。
- Rooms 筛选：当前状态按钮 + 下拉其他选项；Edit 为文字+图标式按钮，编辑态才显示收藏/删除。
- 投票创建：卡片内部为纯色背景，卡片外覆盖层为暗色半透明；按钮、时间、匿名开关都在卡片内部完成。
- 投票展示：选项投票后变成进度条，百分比在选项内部，人数保留在右侧。
- Chat 输入：输入框是固定底部功能条，不随消息数量漂移；语音按钮属于输入框内部控件。
- Board 工具：使用多个独立 icon，不再使用单一加号承载所有操作。
- Board 内容：照片按真实比例，文本标注尺寸随内容和用户调整变化，弹幕评论只显示文本，不加描边框。
- 涂鸦板：全屏，不使用弹窗标题；工具悬浮底部，brush 预览只在调整画笔时出现。
- 覆盖层：Poll History、文本预览、评论输入等使用暗色半透明背景建立层级，但内容卡片本体保持纯色和清晰边界。
