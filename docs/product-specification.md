# EventSpace 第一版产品规格

> 状态：产品决策已确认；所有商业数值保留配置化空间。  
> 范围：移动端优先的私密临时活动空间；桌面端适配；首发英文界面。  
> 参考：完整逐项决策见 `requirements-baseline.md`。

## 1. 产品与边界

EventSpace 让活动组织者创建一个私密、限时的活动房间。房间包含聊天、画板和独立行程三项核心能力；房间结束时由服务器立即冻结并归档。适用派对、婚礼、酒吧、社区活动等多种活动，但产品不提供按活动场景划分的主题或模板。

第一版不提供：公开房间发现流、陌生人私信、全局个人主页、广告、第三方行为追踪、实时位置共享、图片滤镜、画板内容投票、多选/排序投票、站内分摊付款或严格端到端加密。

## 2. 房间生命周期

1. 已登录创建者建立房间，设置名称、可选说明、模式、时长、人数、入口方式、审核、成员/投票可见性、可选封面与地点。
2. 活动期内成员按权限进行聊天、画板和行程协作。
3. 房间可因服务器到期、Host 手动结束或投票结束而冻结。服务器时间是唯一准则；到期无缓冲，所有写操作立即拒绝。
4. 界面先显示“正在归档”。一般数秒完成；媒体较多时后台完成，完成后通知成员。
5. 归档只读，聊天、行程和画板均可查看。照片可以保存到本地；语音可播放但不提供下载按钮。

免费房间最短 15 分钟、最长 24 小时；付费房间最长 7 天。单次最多延长 24 小时，不能缩短到当前时刻之前。无 Host 房间的投票延长也受相同套餐累计上限限制。

## 3. 角色、权限与模式

| 角色/模式 | 核心规则 |
| --- | --- |
| Host | 可结束及修改房间时长、转让 Host、管理管理员、入房、治理与内容。 |
| 管理员 | 几乎拥有 Host 的治理能力，但不能结束、延长或缩短房间。 |
| 已登录成员 | 可投票、上传照片、写画板注释、评论和表情回应；仅编辑自己的内容。 |
| 访客 | 必须提供昵称和头像；可发送基础文字/短语音。登录后继承此前活动记录。 |
| Host 房间 | Host/管理员决定一般管理与行程创建；也可发起投票。 |
| 无 Host 房间 | 创建者完成初始设置后即为普通成员；房间决策由投票产生。模式创建后不可切换。 |

Host 只能转让给房间内已登录成员，接收者确认后生效；原 Host 自动成为管理员。管理员数量不设固定上限，但添加时须显示权限风险提醒。

踢出只将成员移出当前房间；若未被拉黑且入口仍有效，对方可重新申请。拉黑则禁止再次进入。成员被踢出、拉黑或主动离开后立即失去未来访问权；已经看到、复制或下载的内容无法远程撤销。

## 4. 进入、身份与成员资格

- 房间默认私密，不被公开发现。
- 支持链接、链接二维码和邀请码；创建者可设为持链接直接进入、仍需邀请码或必须审核。
- Host 可作废邀请码、轮换链接/二维码。已进入成员保持访问权；新成员必须使用新入口。
- 审核申请展示头像、昵称与申请备注。Host 模式由 Host/管理员处理；无 Host 模式自动形成现有成员投票，超过当前有效成员半数后通过。
- 无 Host 房间的成员名单固定对房间内所有成员可见；Host 模式可设为全员或仅 Host/管理员可见。
- 地理位置限制不在第一版范围内。
- 创建房间必须登录；登录支持 Google 与无密码邮箱 magic link/OTP。
- 访客可上传头像或使用系统抽象头像/昵称首字母；房间内昵称必须唯一。
- 访客关闭浏览器、清除站点数据或换设备后临时身份失效。登录用户可跨设备恢复身份。

归档只对未被踢出的原成员开放，且需要登录。访客若在结束后才登录，只要当前设备仍保存该房间身份，即可绑定个人归档。

## 5. 聊天

- 支持文字、短语音、@ 提及、回复、撤回、置顶和本房间搜索。
- 图片统一上传至画板，不发在聊天区。
- 语音最长 60 秒，支持进度条拖动；不支持转文字或倍速；默认不自动播放。
- 用户可在两分钟内无痕撤回自己的消息；消息不支持编辑。
- Host/管理员可无痕删除其他成员消息；置顶仅限 Host/管理员。
- 输入规则：Enter 发送，Shift+Enter 换行；移动键盘 Enter 同样发送，换行由明确的输入控件完成。
- 不显示在线成员、输入状态或已读回执。
- 消息与评论最长 2,000 字符，支持 Unicode 与 emoji。

系统消息记录成员加入/离开、Host 转让、管理员变更、行程变化、投票发起/结果及房间即将结束；不记录举报与内容删除细节。

## 6. 画板与顺序显示

画板为自由摆放的活动照片与文字注释空间。所有已登录成员可编辑自己的内容；Host/管理员仅能删除他人内容。

- 支持移动、缩放、旋转、编辑文字与删除；不做图片裁剪或滤镜。
- 每张照片首次贴入时采用固定微小倾斜角度、顶部白条与中间图钉的视觉语言；重开房间时角度不可变化。
- 新图片不得覆盖已有图片超过 20%；标注不可被覆盖；文字不可压住图片主体。
- 遮挡冲突时，拖动结束自动吸附到最近可用位置，不显示打断性提示，也不允许强行覆盖。
- 移动端：单指拖动画布/元素、双指缩放、长按选择元素并显示简洁工具条。
- 顺序显示按上传时间展示照片及注释；第一版仅提供时间流与详情查看。
- 画板内容支持登录用户的短评论、回复、@ 提及、表情回应和两分钟撤回。

照片支持 Android/iOS 常用格式与 Live/Motion Photo；不兼容动态照片时转为静态主图并告知用户。原始文件最大 10 MB；上传后去除 EXIF，压缩至最长边 2560px、约 85 质量，不保留原图。移动端支持拍照/相册，桌面支持拖拽上传。

## 7. 行程

行程为独立页面，围绕活动时间线展示开始/结束时间、地点、说明、地图和负责人，不承担成员签到或报名。

- 地点支持地图搜索与手动文字输入；展示地点卡片并跳转 Google Maps/Apple Maps，不做路线或实时位置共享。
- 状态仅为未开始、进行中、已结束，由开始与结束时间相对当前时间自动推导，不提供手动推进或回退。
- Host 房间仅 Host/管理员可创建行程并指定负责人；负责人、Host、管理员可编辑或删除；无 Host 房间行程提案先投票生效。
- 负责人可为访客，以“临时身份 + 当前设备”绑定。
- 创建者设置活动时区（默认设备时区）；数据以 UTC 保存，成员同时看到本地时间和活动时区时间；夏令时自动遵循活动所在地规则。
- 页面按日期纵向排列；每次进入优先滚动到正在进行的行程，没有进行中行程时滚动到下一项或最后一项。

## 8. 投票

无 Host 房间决策通过投票执行；Host/管理员在 Host 房间也可发起同类投票。仅登录用户投票。

- 仅支持赞成/反对事项投票，以及行程方案单选投票。
- 票数始终可见。创建者可设完全公开或完全匿名，默认公开。
- 发起时固定合格成员总数；赞成结果超过半数即生效，不必等截止时间。
- 投票截止时间可手动设置，但不能晚于房间结束时间。
- 无 Host 房间可投票：结束房间、延长/缩短时长、改入口、踢人、禁言、改负责人和行程安排。
- 所有登录成员可发起治理投票；同一成员每 10 分钟最多一次。

## 9. 治理与反滥用

- 每位用户可私密举报。举报人描述违规者及内容位置，Host 私密回复；举报不自动转交原始内容。
- 禁言者仍可浏览、参与行程及投票，但不能发文字/语音/@、评论/表情、照片或画板注释。
- 设备级拉黑使用随机本地设备令牌，不做浏览器指纹；清除站点数据后令牌失效。
- 访客有文字、@ 和语音频率限制；异常时要求简易人机验证。
- 所有写入、权限和到期校验都在服务端执行。

## 10. 归档、账户与付费

归档为个人列表入口：成员只能从自己的列表移除，无法影响其他成员。免费归档保留 30 天；到期前 7 天提醒成员，主存储在到期后 7 天内删除，备份再于 30 天内清理。

用户删除账户时，显示名变为 `Deleted member`、头像移除；内容保留至普通归档到期。永久归档内容保留，但不再关联该账户。

第一版不做个人订阅，所有付费均附着于房间并惠及全部符合归档资格的成员：

- 免费：最多 10 名成员、25 张照片、250 MB 媒体、最长 24 小时、归档 30 天。
- 付费活动扩展：暂定最多 50 名成员、200 张照片、1 GB 媒体、最长 7 天；数值必须配置化。
- 永久归档：独立的一次性房间级项目，与活动扩展分开售卖；按购买时房间容量保存，后续可购买容量扩展包。
- 房间可创建时付费，或在运行中/免费归档未过期时由任一已登录成员升级。首发不做站内分摊。
- 永久归档生效后不退款；支付失败不解锁；付款人删号不影响其他原成员。
- 支付使用 Stripe 托管信用卡支付与基础税务能力，产品不存储支付卡资料。

## 11. 页面与体验

- 房间使用两层固定顶部结构：第一层左侧为返回 Rooms 的图标、中间为严格居中的房间名称/状态、右侧为必要操作；不再显示当前页面指示图标。第二层常驻 `Chat / Board / Itinerary` 三项导航。房间页面只通过导航按钮切换，不提供左右滑动切换；切换时保留各页本地状态与滚动位置。
- 桌面端：聊天页为消息主区与房间信息栏；画板页全宽；行程页为内容区与详情栏。
- 固定房间栏展示名称、倒计时、状态、分享、成员和更多操作，不展示在线人数或付费入口。
- 登录后的首页直接从进行中/已归档切换与房间卡片开始，不显示营销式标题或欢迎语；支持收藏、搜索和筛选。卡片展示画板缩略图、名称、状态、结束时间和成员数。
- 公开首页只提供产品说明、创建、通过链接进入、登录、条款与隐私入口；没有公开房间。
- 网络断线时保留文字草稿，画板操作重连后重试；若权限或状态已改变必须明确告知失败。
- 动效是核心体验：120Hz 设备尽可能接近 120fps，一般设备至少流畅 60fps；性能不足时优先输入、拖动和可读性。遵循“减少动态效果”、键盘操作及色彩对比要求。

首发承诺支持当前及前一主要版本的 Chrome、Edge、Safari（macOS/iOS/iPadOS）；Firefox 不作为首发承诺浏览器。

## 12. 法律与用户支持

- 面向美国、欧盟与澳大利亚的英语用户；最低年龄 16 岁。
- 注册或创建房间须勾选“我已满 16 岁并同意服务条款与隐私政策”，不收集生日。
- 第一版提供支持邮箱或联系表单、服务条款、隐私政策、社区规则、Cookie 说明页面。
- 正式上线前，以上法律文本及地区性隐私/消费者权益流程必须由目标法域律师审阅。
## 2026-07-18 当前同步：产品规格与现状差异

本规格仍作为第一版产品目标；当前代码实现是本地优先 Mock，不代表所有生产能力已经完成。

### 已经在本地版本中具备的产品能力

- Rooms：All / Active / Achieved / Favorite 筛选、搜索、收藏、删除个人入口、真实画板快照预览、移动端单卡横滑与网格切换。
- Create Room：五步创建、移动端时间滚轮、本地账号门槛、创建成功邀请卡片、mock QR / 邀请码、Open this room。
- Chat：底部输入栏、右侧自己消息、发送后滚动到底部、搜索、回复、反应、撤回、置顶、管理员删除、语音条 mock。
- Poll：yes/no、选项、行程投票；open minutes；匿名/公开；结果进度条；投票历史全屏查看。
- Board：无限画布、单指平移、双指缩放、真实图片上传压缩、原图比例显示、文本标注、全屏涂鸦板、弹幕评论、作者信息、删除、Sequence 单列流。
- Itinerary：负责人、起止时间、地点、说明、时间自动状态、冲突提示与外部地图入口；行程投票通过后可添加计划。
- Governance：Host/Admin 与 Community-led 的基础差异、成员审核、禁言、移除、拉黑、入房多数表决、房间结束和归档 mock。

### 当前仍是产品目标、未进入生产实现的能力

- 真实登录、匿名访客认领、跨设备身份、成员邀请发送。
- 多人实时同步、消息送达、冲突处理、服务端裁决。
- 私有媒体存储、EXIF 清理、转码、缩略图、签名 URL。
- 生产级语音上传/转码/跨设备播放、推送、邮件、PWA 离线、支付和永久归档。
- 正式法律文本、用户支持流程、审计日志和反滥用系统。

### 产品规则提醒

当前 UI 已经验证了大量移动端交互，但后端阶段不得把本地 reducer 的结果视为可信事实。房间到期、投票通过、成员资格、内容所有权和归档资格都必须由服务端重新判定。

## 2026-07-19 当前同步：核心体验落地说明

- Chat 的第一版本地体验确定为轻量 Telegram 式操作：消息分组、回复、长按操作、附件托盘、照片、位置、按住录音、置顶和回到最新；不扩展频道、机器人、转发网络、已读回执或复杂联系人体系。
- Board 的体验确定为“共享空间记忆画布”而非专业设计工具：无限平移缩放、照片/便签/涂鸦、本人内容编辑、评论、背景与 Sequence；不引入图层面板、精确属性表和复杂矢量编辑。
- `/rooms/new` 时长仍遵守既有 15 分钟至 24 小时规则；循环滚轮只是移动端选择方式变化，不改变产品配额。
- Host-led 的 Member list 可在 Moderators / Everyone 间选择；Community-led 仍固定 Everyone。此次仅修正控件层级，没有改变权限规则。
- 房间 Board 背景属于共享房间内容，Rooms 卡片必须呈现同一背景和真实 Board snapshot。
